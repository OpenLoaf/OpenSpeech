## v0.2.34

> v0.2.34 是 v0.2.33 的稳定补丁版：修掉一个会让快捷键设置后 UI 整个冻住的死锁；把 AI 整理的提示词体系拆成「内置规则 + 案例 + 按应用覆盖」三层，能针对某一个 App（如 VS Code、邮件、聊天窗）单独配一套整理风格；提示音补齐到所有跨模式切换 / ESC 二段确认场景。

---

### 修复

#### 修改快捷键后 UI 冻住的死锁

- **症状**：在「设置 → 快捷键」改完一个组合按保存后，主窗口 / 悬浮条偶发完全无响应，托盘菜单也卡死，只能强退应用。
- **原因**：内部快捷键应用流程会跨过系统级注册调用持锁等待，与主线程 hotkey 处理器互相等死。
- **修复**：拆成「快照 → 释放锁 → 调用系统 → 重新取锁写回」三段式，全程不再有互相等待的窗口。`pause / resume` 同步修一遍。

---

### 新增

#### AI 整理的提示词改成可分层定制

- **三层结构**：「核心规则」（默认整套出厂规则可逐条勾选启停）+「案例」（你给一组「原文 → 期望整理结果」示范，AI 跟着学）+「按应用覆盖」（为某个具体 App 单独定义一套整理风格）。
- **按应用覆盖**：给 VS Code 配「保留代码风格、不改变量名」、给邮件配「自动加问候语」、给微信配「短句风、保留口语」——同一个用户在不同场景拿到完全不同的整理风格。
- **应用类型自动注入**：识别当前前台应用大类（编辑器 / 终端 / 邮件 / IM / 浏览器…）写进上下文，AI 在没有显式 per-app override 时也能拿出符合该类型的整理风格。
- **历史重转写命中 per-app override**：在历史记录里点重新整理一条旧记录，会按当时录制时所在的 App 命中对应覆盖配置，不是用现在前台应用——避免「重转一条邮件却被当成代码处理」。

#### 提示音补齐

- **跨模式切换有起声**：从普通听写切到翻译听写、或从翻译切回普通时，新模式开始录制时同步播放 start 提示音（之前只在第一次起录有声）。
- **ESC 二段确认进入 armed 阶段有提示音**：录制中按一下 ESC 进入「再按一次取消」的待确认状态时，播放一个短促区分音，让你听到现在到底是「真的在录」还是「正在等你确认要不要丢弃」。
- **ESC armed 提示音调柔**：原版本偏尖锐，调成更轻的提示，长时段会议录制不被打扰。

---

### 改进

- 三套语言（简中 / 繁中 / 英文）同步补齐 AI 整理设置页的所有新文案与按钮提示。
- 内部架构：前端源码迁到独立私有仓，公开仓只保留 Tauri / Rust 部分与构建脚本，下载安装方式不变（`pnpm install` 自动拉取预构建产物，零额外步骤）。

---

### 已知问题 / 暂未做

- 「按应用覆盖」当前只能精确匹配 bundle id / exe 路径，暂不支持通配符或正则；下一版本计划加上。
