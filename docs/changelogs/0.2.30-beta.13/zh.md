## v0.2.30-beta.13

> 测试版，仅 Beta 通道用户可收到。本轮重点是 **输出方式可选** 与 **错误上报真正能上报**——上一版 beta 里的「上报错误」按钮其实被服务端 422 拒了，本版修。

### 新增

- **输出方式 · 两个开关**（设置 → 听写 → 输出方式，默认都开启）
  - **文字逐字输出**：关掉后改成"整段一次粘贴"。微信输入法 / 部分全拼 IME 在逐字注入时会出现首字被吞、双逗号、丢标点等异常——遇到这些 IME 关掉这个开关就能绕开。
  - **自动复制到剪贴板**：关掉后注入完成不留痕。若一次性粘贴路径仍需借用剪贴板，粘贴完会自动还原原剪贴板内容，不会污染用户复制队列。

### 改进

- **统计弹窗 · 图表交互**：每日趋势 / WPM 折线从手写 SVG 换成 recharts，hover 任意柱/点会按指标类型展示原始值（时长用小时分钟、WPM 加 wpm 后缀、字数和次数走数值）。活跃时段热力图与应用 Top 单元格 hover tip 也补上完整 sessions 数。
- **会议错误弹窗 · 长标题换行**：标题改成顶对齐 + 给右上关闭按钮预留空间，长 i18n 文案不会再压到 ✕ 按钮上。

### 修复

- **错误上报 / 反馈意见 422 失败**：OpenLoaf SaaS 反馈接口的 `source` 字段只接受 `tenas | openloaf | openloaf-saas`，之前传的 `openspeech` 被服务端拒，导致 beta.12 加的「上报错误」按钮、设置里的「反馈意见」全都发不出去。本版改回 `openloaf-saas`，客户端身份继续走 `client` 字段。
- **会议错误码 i18n 命中**：之前 `meeting_provider_unsupported` / `meeting_provider_not_configured` 被错误类型 Display 加了一层 `unsupported:` / `unauthenticated:` 前缀，前端按 `^[a-z_]+:` 拆 code 时拿到的是分类名，i18n 命中不到。修后会议错误弹窗能显示真正的人类可读文案。

### 升级提示

- 本版本是 SemVer 预发布段（`0.2.30-beta.13`），仅 Beta 通道下发；Stable 通道用户不会收到。
- 切到 Beta 通道：设置 → 关于 → 更新通道 → 选 Beta，重启或托盘「检查更新」。

### 想让 Beta 用户帮忙验

- **错误上报真的发出去了**：会议弹窗（关网模拟错误）→ 点「上报错误」→ 应当看到「错误已上报，感谢反馈！」的 toast 而不是 422 / 失败提示。
- **微信输入法用户**：在微信聊天框打开/关闭「文字逐字输出」开关，对比注入是否还出现首字吞、双逗号、丢标点。
- **剪贴板隐私**：关掉「自动复制到剪贴板」后做一次注入，确认你之前复制的内容仍在剪贴板，没被这次听写覆盖。
- **统计弹窗 hover**：打开统计弹窗，把鼠标悬停在每日柱图、WPM 折线、活跃时段热力图、应用 Top 行上，确认能看到原始数值（时长、字数、wpm、sessions）。
- **会议错误文案**：故意触发一次会议 provider 错误（如把当前 provider 切到 SaaS / Aliyun / TencentFile 这些不支持实时说话人分离的通道再开会议），弹窗里应当看到中文/英文/繁中三语版本的「当前通道不支持会议转录」之类文案，而不是裸 code。
