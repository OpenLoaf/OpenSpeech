## v0.2.30-beta.0

> 测试版，仅 Beta 通道用户可收到。本轮主线是**翻译听写**——按下翻译快捷键说一段话，转写完成后会被翻译成你设定的目标语言再注入到当前焦点。希望 beta 用户帮忙验各种焦点（输入框 / IM / 浏览器地址栏）下译文是否完整、悬浮条徽章是否扰人。

### 新增

- **翻译听写快捷键**：默认 macOS = `Fn + Shift`，Windows / Linux = `Alt + Shift`。按住说话，松开后转写文本会被译成「翻译目标语言」再注入到当前焦点；自动写一条 `translate` 类型的历史记录。
- **翻译目标语言设置**（设置 → 通用）：英 / 简中 / 繁中 / 日 / 韩 / 法 / 德 / 西，默认英文。zh 用户翻译需求多为译成英文，非中文用户主流也是英文输出，所以默认 `en`，按需在通用设置里改。
- **悬浮条翻译徽章**：录音 / 准备阶段在悬浮条挂一个语言徽章（EN / 中 / 繁 / 日 / 한 / FR / DE / ES），让你一眼知道现在录的这段会被译成什么；进入「思考中」/「输出中」/「错误」后徽章自动消失，避免挤掉中心文案首字。
- **后端日志**：vendor / region / model / engine 等关键参数从 debug-only 改成永久 info 日志，BYOK 通道排查直接看 `RUST_LOG=info` 即可（请求 body 仍只在 debug level 出，不会泄漏到普通日志里）。

### 改进

- **听写模式默认排序**：设置里 `UTTERANCE`（整段优化）放第一位，`REALTIME`（边说边出字）放第二位，与默认值保持一致；新装用户进设置看到的第一项就是默认项。

### 已知风险 / 想让 Beta 用户帮忙验

- 翻译态下**不走 REALTIME 增量注入**，必须等录音结束才出译文。如果你习惯了 REALTIME 边说边出字的节奏，翻译模式下会感觉「按完没反应几秒才出」，这是预期。
- 翻译走的是 chat stream（`OL-TL-006`），网络抖动 / 长录音的失败兜底跟普通 AI 优化共用一条链路，但**翻译 prompt 是新写的**——如果遇到译文里残留了原文、或译完反向把目标语言译回了源语言，请把那段录音 + 译文截图反馈。
- 悬浮条徽章只测了 8 种语言短码；如果你切到了非默认语言看到布局错乱（徽章把波形挤掉一大片），请截图反馈。

### 升级提示

- 本版本是 SemVer 预发布段（`0.2.30-beta.0`），仅 Beta 通道下发；Stable 通道用户不会收到。
- 切到 Beta 通道：设置 → 关于 → 更新通道 → 选 Beta，重启或托盘「检查更新」。
