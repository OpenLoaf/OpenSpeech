## v0.2.32

> v0.2.32 是 v0.2.31 之后两轮 Beta（beta.0 / beta.1）灰度后的稳定正式版。所有 Stable 通道用户都会收到。本版三条主线：**听写结果面板大改**（结果常驻 + 复制/修改/润色/翻译 4 按钮）、**词典联想 + 智能焦点检测让语音识别更准**、**Linux AppImage 重新启用并修真正的启动 root cause**；同时快捷键校验规则全面放宽（单键 / 单修饰键），以及一波稳定性修复。

---

### 新增功能

#### 听写结果面板（重做）

- **结果常驻、不再 10 秒消失**：录音结束后转写结果固定在主面板，直到下一次录音才清空。
- **4 个新动作按钮就在面板底部**——
  - **复制**：一键复制最终文本（已润色 / 已翻译都跟随当前显示态），点完按钮变绿勾 1.5 秒确认。
  - **修改**：直接弹历史编辑对话框改这条文本，diff 实时高亮。
  - **润色 · 场景**：右侧下拉切换润色场景（学术 / 邮件 / 转写整理 / 自定义场景等），点击按钮流式润色，结果就地替换主显示文本。润色中按钮显示「润色中…」并 pulse。
  - **翻译 · 语言**：右侧下拉切目标语言（与设置 → 翻译目标语言双向同步），点击按钮流式译出，原文 + 译文分组并列展示。
- **耗时统计上挪到顶部**：「录音 X.Xs · 响应 Y.Ys」（响应 = ASR + AI refine 合并），让底部按钮区更清爽。
- **流式期间也能看到中间稿**：润色 / 翻译流式输出时主面板实时滚动展示部分结果，不用等全部完成。

#### 词典联想（dictionary agent）

- **跨领域高频专有名词自动联想**：在你输入或听写时，系统自动匹配 Cursor / DeepSeek / Llama / Qwen 等几百个专有名词，作为热词送入语音识别提高同音命中率，特别是科技、产品、品牌名等容易被识别成同音错字的词。

#### 智能焦点检测（focus check）

- **AI 整理按当前活跃窗口语境分流**：识别你当前焦点所在的应用（VSCode / 微信 / Notion / 浏览器等），让 AI 整理时更贴合你正在写的东西。**跨应用历史不再互相污染**——在 VSCode 写代码注释和在微信聊天的整理风格自动区分。
- **macOS 焦点可编辑检测时机改进**：从「录音开始时 snapshot 一次」改成「准备 inject 的瞬间当场查」。录音开始时焦点不可信（用户可能录到一半才点进输入框，web 表单的 textarea 直到聚焦才被 AX 识别为 editable），改在写字瞬间问最准。silent fail（enigo 在 Finder/桌面 / 只读 web 区域 type 不抛错也写不进字）至此根治：拿到不可编辑结论时直接跳 inject，走结果面板让你手动复制。

#### 历史记录

- **历史记录支持手动编辑**：在 History 页可以直接编辑某条转写文本，对话框内会高亮 diff，让你看清楚改了什么；新结果面板的「修改」按钮也直接弹这个对话框。
- **目标应用名按界面语言本地化**：之前历史里只显示进程名（`WeChat`、`weixin`、`钉钉` 各自一份），现在按当前界面语言归一显示——简中显示「微信 / 钉钉 / 飞书 / 腾讯会议 / 终端 / 访达」等；繁中显示「釘釘 / 飛書 / 騰訊會議 / 終端機」；英文显示「WeChat / DingTalk / Lark / Tencent Meeting / Terminal / Finder」。收录 30+ 中文常用 app + macOS 系统应用；不在表里的应用按系统给的原名展示。

#### 快捷键

- **校验规则全面放宽**——
  - **`combo` 允许单按主键**（如单按 `F8` / `Home` / `End`）：旧规则强制至少 1 个修饰键，新规则放开。
  - **`modifierOnly` 最少 1 个修饰键**（之前必须 ≥ 2 个）：单 `Fn` / 单 `Option` 等单修饰键听写终于可以绑了。
  - 新增 **W3 软警告**：单修饰键 modifierOnly 会吞该键的所有原生组合（Alt+Tab / Win+E / ⌘+Space / ⌥+Arrow 等），录入时按平台给出针对性提示。
  - **F1-F12 单按软警告只在 macOS 出现**：Windows / Linux 上 F-key 直接作为 F-key 触发，不冲突，不再唠叨。
- **新增硬拦**：
  - 单按"输入键"（A-Z / 0-9 / Space / 主键盘标点 / Numpad 数字）必须配修饰键——否则每次正常打字都会触发快捷键，成本极高。
  - `NumLock` / `ScrollLock` 加入 combo 永久禁用主键列表（与 `Tab` / `Backspace` / `Delete` / `CapsLock` 同级）：锁定类键的 release 在多平台都不可靠。

### 改进

- **AI 整理与语音识别共用同一份上下文规则**：合并到 `buildSpeechSystemPrompt`，整理流程更稳；按目标应用隔离历史，跨应用噪声不再污染整理结果。
- **AI refine 系统提示词重构**：按 2026 prompt engineering 指南重写，输出更稳定。
- **会议实时听写后端重构**：`saas.rs` 大幅瘦身（- 576 行），长会话更稳更流畅。
- **机器信息（platform / 时区 / 系统语言）一并送入上下文**，让模型在多语种混说时表现更自然。
- **波形条数从 20 加到 28**（柱宽 4px → 2px）：单根更细、整体更密，整个波形窗口约 1.4s 就能看全一句话节奏。
- **Toolbox 录音上下文判定**：录音启动那一刻检测 Toolbox 主窗口是否聚焦+可见，未聚焦则不把转写结果自动写到 Toolbox 输入区——避免「Toolbox 在后台，前台用全局快捷键听写到别的应用，结果反而被塞进 Toolbox 输入框」。
- **升级 OpenLoaf SaaS SDK**：0.3.18 → 0.3.19。
- **dictionary toast 在 dark 主题下颜色修**：sonner 默认把 description 颜色硬编码成亮灰，在 te-accent 黄底上看不清；强制继承 te-accent-fg。

### 修复

- **Linux AppImage 启动后立刻退出**：自定义 desktop entry 的 `Terminal=true` 误触发，硬编码为 `false` 修复（v0.2.31 是 linuxdeploy 工具链镜像过期完全没打出 AppImage；本版彻底修好启动 + OTA 自动更新链路）。
- 单按 F-key 在 macOS 之外不再误报"系统功能键冲突"（W2 限定 macOS）。

### 升级提示

- 本版本是 stable 正式版（`0.2.32`），所有 Stable 与 Beta 通道用户都会收到。
- **Linux 用户重新拿到 AppImage**：v0.2.31 因 Tauri 上游 linuxdeploy 镜像过期暂停 AppImage，本版 AppImage / deb / rpm 三件套同时发，OTA 自动更新恢复。
- **快捷键规则变更不影响已有 binding**：原本绑了「2 个修饰键的 modifierOnly」/「带修饰键的 combo」继续生效。新规则只是把"以前不允许的录入"放开了。
- 老 beta 用户（已在 v0.2.32-beta.0 / beta.1）无需重装，自动 OTA 升级到 v0.2.32 stable。
