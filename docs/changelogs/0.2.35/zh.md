## v0.2.35

> v0.2.35 是 v0.2.34 的稳定补丁：修掉一个 macOS 下偶发出现的「按了 PTT 没反应 / 转录失败」根因——输入设备列表枚举走 CoreAudio 慢路径会卡 3-10 秒，挂在录音预检里把整条按键链路拖死。

---

### 修复

#### 「按了 PTT 没反应 / 转录失败」（macOS 偶发）

- **症状**：按下听写快捷键后键盘像没响应，或者完成后 toast 提示「转录失败」，反复几次才能成功录一次。多在蓝牙耳机刚连/刚断、装了 OBS / BlackHole / Loopback 这类虚拟声卡、或者系统刚从睡眠唤醒时出现。
- **原因**：每次按 PTT 都要先枚举一遍系统输入设备列表来做预检，这个调用在 macOS 上经 CoreAudio HAL，蓝牙重连/虚拟声卡/`coreaudiod` 抖动时偶发卡 3-10 秒。期间用户以为没响应又按一次，第二次按下被状态机当成「再按一次结束」，但 mic 还没真起来，录到一段几乎为空的音频送去转写，自然拿不到文字，弹出「转录失败」。
- **修复**：设备列表加 cache，启动期间后台预热一次；之后预检直接读 cache、微秒级返回。即使 CoreAudio 真的卡，cache 也会返回上一次的快照不阻塞按键。
