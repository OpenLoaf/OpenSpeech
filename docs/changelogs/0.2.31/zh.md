## v0.2.31

> v0.2.31 是 v0.2.29 之后累积近两个月、17 轮 Beta 灰度的稳定正式版。所有 Stable 通道用户都能收到。本版主线有四条：**翻译听写**、**会议转录**、**快捷键系统重构**、**AI 改写体验大改**；同时大量稳定性修复（Windows 注入、唤醒登录、网络抖动等）。
>
> （v0.2.30 是 v0.2.31 的早期内部构建，未对外发布；从 v0.2.29 升级会直接收到 v0.2.31。）

---

### 新增功能

#### 翻译听写

- **专属翻译快捷键**：录一段语音 → 自动转写 + 翻译 + 注入到当前焦点，全程不离开键盘。默认 macOS = `Fn + Shift`、Windows = `Alt + Win`、Linux = `Alt + Super`。
- **8 种目标语言**：英 / 简中 / 繁中 / 日 / 韩 / 法 / 德 / 西。设置 → 通用 → 翻译目标语言切换。默认英文。
- **两种输出形态**：
  - `仅目标语言`（默认）—— 只注入译文，最快
  - `双语` —— 先注入清洗后的中文整理稿，空行后接译文
- **悬浮条翻译徽章**：录音 / 准备阶段挂语言徽章（EN / 中 / 繁 / 日 / 한 / FR / DE / ES），一眼确认这段会被译成什么。
- **悬浮条「正在翻译中…」状态**：refine → translation 两阶段独立进行，「思考中 / 输出中 / 翻译中」分开展示。

#### 会议转录

- **新页面：会议转录**（左栏新入口，麦克风图标）—— 长录音、自动按发言人分段、时间轴回看。
- **两种实时 ASR 通道**：
  - **OpenLoaf SaaS**（推荐，新增）—— 鉴权走当前登录账号，扣费走 SaaS 积分，开箱即用，无需配密钥
  - **腾讯云 BYOK**（速度最快、支持说话人分离）—— 配 AppID / SecretId / SecretKey 后启用 `16k_zh_en_speaker` 引擎
- **AI 纪要（流式）**：会议结束/进行中点「生成纪要」，AI 流式产出 Markdown 格式纪要（决策、行动项、关键讨论点），落盘到本地，重启后仍能打开。系统提示词三语可在「设置 → AI → 系统提示词 → 会议纪要」自定义。
- **导出 Markdown**：会议详情页右上「导出」→ 系统保存对话框 → 写入元数据头 + 按发言人分段的 Markdown 文件。
- **真实波形**：详情页支持点击 segment 跳转到对应时间；live 录制中实时画波形（暂停时冻结最后一帧而不清零）。
- **断网自动重连**：网络抖动会触发指数退避重连（0.5s → 1s → 2s → 4s → 8s × 5 次），重连期间顶部出现琥珀色提示横幅；句子 ID / 时间轴接续不会错乱。
- **错误码引导对话框**：遇到鉴权失败 / 配额不足 / 网络异常时弹清晰对话框，告诉你下一步去哪改 + 提供「上报错误」按钮（一键匿名上报错误码 + 平台 + 版本 + 日志尾巴到 OpenLoaf 反馈通道）。
- **历史记录支持 `meeting` 类型**：每条 final 片段独立入库（sentence_id / speaker_id / start_ms / end_ms / text），详情页可按发言人时间轴回放。

#### 快捷键系统大重构

- **4 个 binding 默认就位**：
  - `dictate_ptt`（听写）
  - `translate`（翻译听写）
  - `show_main_window`（唤起主窗口）—— `Ctrl + Alt + O`
  - `open_toolbox`（打开 AI 工具）—— `Ctrl + Alt + T`
- **左右修饰键区分**：左 Ctrl ≠ 右 Ctrl 等按物理键判定，预览和触发都按物理键来。fn 不分左右。旧绑定升级时统一记为「左」，习惯用右侧修饰键的可在设置 → 快捷键里重录恢复。
- **冲突类型识别 + 弹窗提醒**：录入快捷键时自动检测多种冲突并清楚说明原因——
  - Modifier-only 互为子集（"会出现幻影录音"）
  - Modifier-only 与 Combo 子集（"按下时会双触发"）
  - 双击与共用修饰键（"双击窗口会冲突"）
  - 系统快捷键预警（如 Cmd+Q / Ctrl+C）
  - F1–F12 单独绑定时提示「在多数 Mac 默认是亮度 / 音量等系统功能键」
- **录入校验扩展**：Modifier-only 至少 2 个修饰键、Tab/Backspace/Delete/CapsLock 不能作为主键、Fn + 主键不可组合、macOS 双击 Fn 拦截。
- **键帽可视化**：HotkeyBinder / HotkeyPreview 用统一键帽样式渲染，左右修饰键带角标。
- **启动自检**：从老版本升级上来的 binding 命中新规则时启动会弹 HotkeyConflictDialog 让你逐条重录，避免静默失效。
- **「全部恢复默认」入口**：设置 → 快捷键页面右上角。

#### AI 改写体验

- **领域多选（最多 3 个）**：词典页面新增 16 个领域可选——`programming` / `ai_ml` / `cybersecurity` / `design` / `film_production` / `photography` / `marketing` / `finance` / `law` / `academic` / `medicine` / `tcm` / `psychology` / `engineering` / `automotive` / 等。勾选后作为 prompt 头部的领域标签注入，让 AI refine 在你的专业语境下更准（例如选 `medicine` 后医学专业名词不再被改成"通俗近义词"）。多语言显示，切语言不丢勾选。
- **默认提示词全套重写**：按 2026 prompt engineering 指南重新拉了听写优化 / 翻译 / 润色 / 会议纪要四类系统提示词，输出更稳定，不再"过度润色"。
- **AI 整理 · 思考过程**：默认提示词注入 `<thinking_process>` 块，模型整理前会先判断"整段语境是什么 / 末尾是不是被切断了 / 哪些口语数字应当转写成阿拉伯数字"——
  - 中文口语数字 → 阿拉伯简写（「六六六」→ `666`、「二零二四」→ `2024`）
  - 末尾被切断音节剔除（「准确度还是比较高的，比。」末尾的「比」会被丢掉，不会硬补成完整词）
- **提示词独立 dialog**：四类系统提示词从内嵌 textarea 改成独立 dialog，主设置页只显示「已自定义 / 默认」状态徽章和「修改」按钮，整页大幅瘦身。
- **润色场景独立 dialog**：润色场景列表也收进 dialog，主页面只显示一行 hint 摘要。
- **历史上下文截断**：历史 entry 拼进 AI prompt context 时按 500 tokens 上限尾部截断 + 标「省略 N 字」，不再爆 context。

#### 听写稳定性 · BYOK 体验

- **录音首尾静音自动裁剪**：松开按键后跑离线 webrtc-vad + 二级 RMS 能量门找首尾 voice 边界 + 300ms padding，前后没说话的部分不再进入录音文件 / ASR / 历史；整段无人声直接跳过转写，不留空 history 条。
- **录音提示音**：开始 / 停止 / 取消三种动作各一个低音量短提示音（可在设置 → 录音里关闭）。
- **唤醒后被踢登录修复**：电脑睡眠唤醒后第一次按下听写不会再被弹回未登录态。两道防线——请求前预检 token 寿命，距过期 ≤ 30 秒先静默续期；万一已过期碰到 401，自动续期重发。
- **401 自动续转写**：录音结束后云端转写撞 401 会自动弹 LoginDialog，登录回来后**自动用刚才的录音重新转写一次**，结果在专属 dialog 里展示——用户不用重录音。
- **BYOK 易用性大改**：
  - 自定义供应商加「已验证 / 未验证」状态徽章，改了任何会影响请求的字段自动重置为未验证
  - 关闭设置时若 active provider 没测通过，自动回退到云端模式 + toast 解释
  - 未配齐的自定义通道改为静默 fallback 到云端（区别于"配过但 keyring 缺凭证"——后者仍会抛错避免悄悄掩盖配置问题）
  - BYOK 失败的 toast 多了「切到云端」按钮，一键回退
- **输出方式可选**（设置 → 听写 → 输出方式，默认都开启）
  - **文字逐字输出**：关掉后改成"整段一次粘贴"。微信输入法 / 部分全拼 IME 在逐字注入时会出现首字被吞、双逗号、丢标点等异常——遇到这些 IME 关掉就能绕开。
  - **自动复制到剪贴板**：关掉后注入完成不留痕，原剪贴板内容不被覆盖。
- **Windows 长段中文注入分块**：中文一字 = 2 个 UNICODE 事件，整段一次性灌进 SendInput 时目标应用 message pump 跟不上会丢字；改成按 4 字符分块 + 块间 sleep 8ms。
- **Windows 中文 IME 吃 ASCII 修复**：中文输入法状态下用「翻译」快捷键输出英文/数字时，注入前会主动清空 composition 再写入，不再首字被吞、双逗号、丢标点。
- **录入快捷键无延迟生效**：以前点「录入快捷键」按钮后会先有一段 ~150ms settling 延迟才开始监听按键，期间预先按下的修饰键会被吃掉；现在立刻挂监听器，按下即响应。
- **悬浮条状态机重排**：录音 / 转写 / 改写各阶段切换更连贯，少了几处状态错位导致的"卡在 loading"。

#### 工具箱

- **粘贴板检测**：粘贴板有内容时给「粘贴并翻译」「仅粘贴」两个快速入口。
- **目标语言扩到 8 语**：与翻译听写一致。
- **运行结果区**加耗时与字数 metadata footer。

#### 词典

- **快速添加 dialog**：QuickDictDialog，可在听写 / 工具箱场景一键唤起添加词典 + 选领域，不必跳到设置页。

#### 历史记录

- **列表无限滚动**加载更多 + 「已到底 · 共 N 条」尾标。
- **详情页可单独切换查看「译文」视图**（仅翻译听写记录）。
- **复制调试提示词**：详情页操作菜单新增「复制调试提示词」，把当前条目的原始 ASR + refine prompt 一键复制走，方便外部对照测试。
- **录音瞬间抓前台应用名**写入 history（如 Chrome / VSCode / 微信），为后续按应用看用量统计打底。

#### 用量统计

- **新增「查看本月用量」入口**（设置 → 关于）。
- **6 类 KPI 维度扩展**——
  - 时长 → 录音时长分桶（< 10s / 10–30s / 30–60s / 1–2min / > 2min）+ 活跃时段热力图
  - 字数 → session 长度分布 + 类型混合（听写 / 提问 / 翻译）
  - WPM → 模式 vs 平均 + Provider 分布（Tencent / Aliyun / OpenAI / SaaS …）
  - 节省时间 → 等价时长 + AI 改写覆盖率
  - Sessions → 状态混合（成功 / 失败 / 取消）+ ASR 通道分布
- **图表交互升级**：每日趋势 / WPM 折线换成 recharts，hover 任意柱/点显示原始值（时长 / 字数 / WPM / sessions）。
- **数据接通真实历史**：之前的 demo 假数据已下线；今日维度按小时桶展示。

#### 国内更新分发

- **更新分发改为 Cloudflare R2 + 腾讯云 CDN，按地区自动分流**：国内用户走 `openspeech-cdn.hexems.com`（腾讯云 CDN 回源 R2），海外用户走 `openspeech-r2.hexems.com`（R2 自定义域名）。每条通道都有异地 host + GitHub Release 三层兜底——「检查更新 / 下载更新」明显更快，不会再"卡在 5xx 拿不到"。

### 改进 · 视觉 / UI

- **主页 Hero 收紧**：移除底部 PUSH-TO-TALK / 跨平台 tagline 一行；快捷键卡片占比加大，焦点更落在快捷键演示上。
- **听写 / 翻译 Tab 标题**：去掉"快捷键"三字冗余，缩为「听写 · PUSH-TO-TALK」「翻译 · TRANSLATE」。
- **桌面快捷键卡片 · 听写状态徽章**：听写 Tab 标题旁直接显示「分段模式（实时 / 整句）」与「AI 优化是否启用」两个状态条，不用进设置就能确认当前模式。
- **悬浮窗模式提示**：模式切换 toast 带 mic / languages 图标，听写态 → mic、翻译态 → languages，瞄一眼图标就能确认。
- **AccountDialog 体验修**：长邮箱不再撑破对话框；微信登录后昵称正常显示；不同账号信息长度的对话框高度跟着内容走。
- **设置页大幅瘦身**：AI 提示词、润色场景全部收进独立 dialog，主页面只显示状态徽章。
- **历史详情面板瘦身**：调试信息折叠到次级菜单，主信息区更突出。
- **首屏闪烁修**：i18n 偏好读取改 await，避免主窗先按 navigator.language 渲染再切到用户偏好的"闪一帧"。
- **i18n 三语全量同步**：所有新文案在 zh-CN / zh-TW / en 都已对齐。

### 修复

- 录入快捷键时按下非法组合给出更具体的错误（modifier_only_min_two / combo_disallowed_main / combo_no_fn / double_tap_fn_macos 等），而不是笼统的「非法组合」。
- show_main_window / open_toolbox 等窗口指令在主窗口已聚焦时会短路，不再重复触发动画。
- 错误上报 / 反馈意见 422 失败修复。
- 会议错误码 i18n 命中修复（不再显示裸 code）。
- 快捷键 modifier-only 绑定在某些机型释放时偶尔丢事件的小毛病。
- 会议页在网络抖动后偶发的"假阳性 VAD 截断"。

### 升级提示

- 本版本是 stable 正式版（`0.2.30`），所有 Stable 与 Beta 通道用户都会收到。
