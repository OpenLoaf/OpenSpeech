## v0.2.30-beta.9

> 测试版，仅 Beta 通道用户可收到。本轮是 beta.8 的稳定性补丁：**修复电脑睡眠 / 长时间不用后再用 OpenSpeech 会被退出登录**。其余沿用 beta.8。

### 修复

- **唤醒后被踢登录**：电脑睡眠期间 SaaS 后台续期定时器停跑，唤醒后第一次按下听写 / 翻译 / 短音频转写时，请求带的是已经过期的 token，服务端返 401，原本会把这次 401 当作"登录彻底失效"立即把用户清场登出。本版改成两道防线：
  - **请求前预检**：访问 SaaS 之前主动检查 token 寿命，距离过期 ≤ 30 秒就先静默续期再发请求；
  - **401 自动续期重发**：万一已经过期碰到 401，先尝试续期 + 重发一次，重发还失败才视为真正登录失效。

  覆盖三条直连 SaaS 的链路：实时 ASR 听写、saas-file 短音频转写（≤ 5 min）、AI 文本改写 / 翻译（chat completions）。saas-file 长音频（> 5 min）入口处也补了预检。

### 排查 / 调试

- 上述路径在过期 / 命中 401 / 续期 / 重发的关键节点都补了日志，万一再出现"莫名被登出"，可以从日志里看清楚是「续期成功重发拿到结果」还是「续期也失败真的过期」。

### 升级提示

- 本版本是 SemVer 预发布段（`0.2.30-beta.9`），仅 Beta 通道下发；Stable 通道用户不会收到。
- 切到 Beta 通道：设置 → 关于 → 更新通道 → 选 Beta，重启或托盘「检查更新」。

### 想让 Beta 用户帮忙验

- **唤醒场景**：合上 Mac 盖子或锁屏一段时间（≥ 1 小时，确保 token 已经过期），重新打开后立刻按听写快捷键说话。预期：能正常转写出来 + 帐号还在线，不会弹回未登录态。
- **AI 翻译 / 改写**：唤醒后第一次触发翻译听写 / Toolbox 改写。预期：和上面一样不会被踢登录。
