## v0.2.17

### 改进

- 录音冷启动稳定性。之前在 macOS 冷启动期偶发「按下快捷键但没开始录音」——根因是麦克风 stream 初始化要 ~1 秒，但前端只等 300 ms 就开 STT 会话，撞 `audio stream not running`。改为 Rust 端同步等 stream 真正起来才返回（上限 1.5 秒），前端按结果决定是否继续，失败会明确提示「麦克风未就绪」。
- 波形噪声门。开车 / 风噪 / 空调底噪不再把悬浮条波形拉满——低于约 −28 dBFS 的窗口直接送 0，常见车厢 / 风扇 / 键盘机械底噪被门挡住，肉眼看到的是「不说话就贴底」。
- 录音中按 Esc 取消时，前台 app 不再被 Esc 误触发。之前在 Cursor 里 Esc 取消录音同时退出了 vim 模式 / 关掉了 IME 候选窗——现在录音活跃期 Esc 被全局拦截，只触发取消逻辑。
- 修饰键漏 release 兜底。Cmd+Tab / 切换空间偶发漏接键释放事件，残留状态会让用户感觉「必须按两次快捷键才有反应」。新增一致性检查，发现 press/release 与内部状态对不上时立即重置，下一次按下必然 emit。
- 没说话 / 全程静音不再弹「无识别结果」提示，悬浮条静默关闭（历史记录仍按失败状态保留）。
- 试用引导页波形修复：之前只有右侧 25% 显示真实波形，左侧一直贴底；现在均匀铺满整条。
- DMG 安装界面终于按设计稿显示了：540×380 紧凑窗口 + 自定义品牌引导背景（金色箭头从 OpenSpeech.app 指向 Applications）。之前 0.2.15 / 0.2.16 因 CI 在 headless 环境下没法让 Finder 写 dmg 视图配置，看到的是 macOS 默认 660×400 排列；这次给 CI 加了 Finder warm-up 步骤，下次开 dmg 就是新外观。

### 修复

- 关闭悬浮条时偶现「全黑长条一闪」：fade-out 动画结束后到窗口物理隐藏之间存在小窗口，黑底 overlay 在那段时间会被看见。改为状态切到关闭的同一帧立即隐藏窗口。
- 取消快捷键单击 ARM 阶段不再弹一闪而过的提示横条（实测无价值，反而干扰）。
