## v0.2.30-beta.3

> 测试版，仅 Beta 通道用户可收到。本版本主要是**分发链路迁移**（用户可感知 = 国内更新检查 / 下载更快）以及若干 UX 修复。

### 改进

- **更新分发改为 Cloudflare R2 + 腾讯云 CDN，按地区自动分流**：国内用户走 `openspeech-cdn.hexems.com`（腾讯云 CDN 回源 R2），海外用户走 `openspeech-r2.hexems.com`（R2 自定义域名）。每条通道还多一层异地 host + GitHub Release 三层兜底。客户端按系统 locale 信号判断 CN / 海外，错判代价仅是"慢一档"，不会拿不到更新。
- **录入快捷键无延迟生效**：以前点「录入快捷键」按钮后会先有一段 ~150ms settling 延迟才开始监听按键，期间预先按下的 Fn 等修饰键会被吃掉，导致用户感觉"卡住"直到 5 秒超时。现在立刻挂监听器，按下即响应。
- **AI 提示词重写**：内置默认 refine 提示词整体重写，更短更聚焦，错改率下降。如果你之前自定义过 prompt 不会被覆盖。
- **听写默认排序**：设置 → 听写设置 里 `REALTIME`（边说边出字）排到 `UTTERANCE`（整段优化）前面，与默认推荐保持一致。
- **后端日志**：腾讯云 COS 转写预签名 URL 进 debug 日志，BYOK 链路排查更方便。

### 兼容性

- ≤0.2.30-beta.2 的老客户端继续从腾讯云 COS 拉 manifest（manifest 已指向新 R2 域名），可以正常升到 beta.3。
- 0.2.30-beta.3 起，新客户端走 R2 + CDN 分发链路，COS 仅保留 manifest 兜底，预计两到三个 stable 版本后完全下线。

### 已知风险 / 想让 Beta 用户帮忙验

- **国内用户**首次收到更新检查走 CDN，第一次 MISS 第二次 HIT；如果发现「检查更新」很慢或 5xx，请把 OS / 城市 / 时间反馈。
- **海外用户**直连 R2 自定义域名，如果遇到下载中断 / 慢，把日志里 `openspeech::updater` 那几行截图反馈。
- 录入快捷键修复后请用「Fn+空格」「Alt+Shift+Z」之类组合多按几次确认捕获正确。

### 升级提示

- 本版本是 SemVer 预发布段（`0.2.30-beta.3`），仅 Beta 通道下发；Stable 通道用户不会收到。
- 切到 Beta 通道：设置 → 关于 → 更新通道 → 选 Beta，重启或托盘「检查更新」。
