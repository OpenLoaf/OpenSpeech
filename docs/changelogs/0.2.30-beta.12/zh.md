## v0.2.30-beta.12

> 测试版，仅 Beta 通道用户可收到。本轮重点：**会议转录的稳定性 + 错误上报通道**、**统计弹窗接通真实数据**、**AI 整理提示词新增「先想再写」思考过程**。

### 新增

- **错误一键上报**：会议转录失败弹窗左下角新增「上报错误」按钮，一键把错误码、平台、版本和最近日志尾巴匿名上报到 OpenLoaf 反馈通道，遇到「登录失效 / 网络异常 / 请求超时」时给出针对性提示。
- **AI 整理 · 思考过程**：默认提示词（中/英/繁中）注入 `<thinking_process>` 块，模型整理前会先判断"整段语境是什么 / 末尾是不是被切断了 / 哪些口语数字应当转写成阿拉伯数字"。新增两类示例：
  - 中文口语数字 → 阿拉伯简写（「六六六」 → `666`、「二零二四」 → `2024`）
  - 末尾被切断音节剔除（「准确度还是比较高的，比。」末尾的「比」会被丢掉，不会硬补成完整词）
- **桌面快捷键卡片 · 听写状态徽章**：听写 Tab 标题旁直接显示「分段模式（实时 / 整句）」与「AI 优化是否启用」两个状态条，不用进设置就能确认当前模式。

### 改进

- **会议转录 · 实时识别更稳**：腾讯 16k_zh_en_speaker 通道在多说话人嵌套时不会再丢句；worker 改成独占 session、音频走 channel 投递，停止与重启之间不会再卡住；时间轴落盘改 jsonl，详情页可以直接回放原音并跳转对应字幕。
- **统计弹窗 · 真实数据**：弹窗里所有图表（每日趋势 / 活跃时段热力图 / 应用 Top 6 / WPM 分布 / 短中长会话占比）全部接通本机历史，今日维度改用按小时桶展示；之前的 demo 假数据已下线。
- **悬浮窗模式提示**：模式切换 toast 现在带 mic / languages 图标，听写态 → mic、翻译态 → languages，瞄一眼图标就能确认。

### 修复

- **Windows 中文 IME 吃 ASCII**：中文输入法状态下用「翻译」快捷键输出英文/数字时，前几个 ASCII 字符偶发被 IME 吞掉；注入前会主动清空 composition 再写入，避免首字被吞或出现双标点。

### 升级提示

- 本版本是 SemVer 预发布段（`0.2.30-beta.12`），仅 Beta 通道下发；Stable 通道用户不会收到。
- 切到 Beta 通道：设置 → 关于 → 更新通道 → 选 Beta，重启或托盘「检查更新」。

### 想让 Beta 用户帮忙验

- **会议转录稳定性**：开一场较长会议（>30 分钟、多人交替发言），确认中途暂停 → 继续不会卡住、不会丢句；离开详情页再进来仍能回放。
- **错误上报**：故意制造一个会议错误（关掉网络后开会议），点弹窗里的「上报错误」按钮，确认 toast 给的提示符合状态（无网时应是「网络异常」而不是泛化失败）。
- **AI 整理 · 中文口语数字 / 切尾**：录一段「六六六，现在准确度还是比较高的，比」这种带切尾的中文，确认输出整理成 `666，现在准确度还是比较高的。`（去掉孤立的「比」）。
- **统计弹窗 · 真实数据**：使用一两天后打开统计弹窗，确认「每日趋势 / 活跃时段 / Top 应用」对得上你实际的使用情况；把范围切到「今日」时主图应改成按小时桶。
