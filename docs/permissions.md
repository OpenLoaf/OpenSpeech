# 平台权限规则

OpenSpeech 依赖系统级权限才能完成"录音 + 全局快捷键 + 文本注入"。本文档定义各平台的权限要求与用户引导规则。

## 所需权限

| 权限 | 用途 | 必需 |
|---|---|---|
| 麦克风 | 录音 | ✅ |
| 辅助功能 / 可访问性 | 监听全局快捷键、向其他应用注入文字 | ✅ |
| 自启动 | 用户开启"开机自启"后才需要 | 可选 |
| 通知 | 错误提示、更新提示 | 可选 |

## macOS

| 权限 | 授权路径 | 必需 |
|---|---|---|
| 麦克风 | 系统设置 → 隐私与安全性 → 麦克风 | ✅ |
| 辅助功能（enigo 键盘注入、读取前台应用名） | 系统设置 → 隐私与安全性 → 辅助功能 | ✅（未授权则注入降级为剪贴板） |
| 输入监控（rdev 全局键盘监听） | 系统设置 → 隐私与安全性 → 输入监控 | ⚠ 推荐（不给则部分兜底机制失效，如 PTT 松开心跳） |

规则：
1. 首次启动通过 [onboarding.md](./onboarding.md) 的 Step 2 引导麦克风授权；Accessibility 和 Input Monitoring 在此页也给出"前往授权"入口，但允许延后。
2. 任一权限在运行中被用户撤销时：
   - 麦克风被撤销 → 下一次快捷键触发时立即进入 Error 状态，悬浮条提示并给出直达系统设置按钮。
   - Accessibility 被撤销 → 注入前用 `AXIsProcessTrusted()` 预检，失败则自动降级剪贴板 + Toast 提示，主界面同时挂黄色横条直到恢复。
   - Input Monitoring 被撤销 → rdev 线程静默失效；主界面挂黄色横条。
3. 用户误点"拒绝"时提供重置步骤：展示 `tccutil reset Microphone com.openspeech.app` 等命令（按界面截图 + 一键复制提供）。
4. `Fn` 键需通过输入监控权限 + `CGEventTap` 捕获；**MVP 不支持 Fn 作为快捷键修饰**，作为后续进阶能力。

## Windows

| 权限 | 说明 |
|---|---|
| 麦克风 | 设置 → 隐私 → 麦克风，允许桌面应用访问 |
| 无 UAC 要求 | 全局快捷键与文本注入无需管理员权限 |
| Defender | 首次运行可能触发 SmartScreen 警告；需对安装包进行代码签名以消除 |

规则：
1. 首次启动若麦克风被系统拒绝，给出直达"Windows 隐私设置"的按钮。
2. **UIPI 注意事项**：OpenSpeech 默认以普通用户权限运行。若目标应用是"以管理员身份运行"（如 cmd、部分编辑器），Windows UIPI 会静默阻止 `enigo` 的键盘模拟。检测方式：比较目标窗口进程的 IntegrityLevel 与自身；若目标更高则：
   - 自动降级为"写入剪贴板"，Toast 提示"目标应用权限高于 OpenSpeech，请手动 Ctrl+V 粘贴"。
   - 同时提供"以管理员重启 OpenSpeech"按钮（一键 relaunch with elevation prompt）。
3. 不默认要求管理员；若用户曾以管理员启动过，下次启动保留状态并在设置里提示。

## Linux

权限模型与桌面环境强相关：

| 子系统 | X11 | Wayland |
|---|---|---|
| 全局快捷键 | ✅ 直接支持 | ⚠️ 依赖 `xdg-desktop-portal-global-shortcut`（KDE/GNOME 较新版本） |
| 文本注入 | ✅ XTest | ⚠️ 需要 `ydotool` / `uinput` 或 Portal API |
| 麦克风 | PulseAudio / PipeWire | PipeWire |

规则：
1. 首选发布 **AppImage + DEB + RPM** 三种包；可选 Flatpak。
2. **MVP 仅保证 X11 下完全可用**；Wayland 下若 Portal 不可用，则：
   - 启动时检测 `org.freedesktop.portal.GlobalShortcuts` 接口可用性；不可用则禁用快捷键设置 UI（字段整行灰掉 + 提示 "当前桌面环境不支持全局快捷键"）。
   - 剪贴板注入优先 `wl-copy`，粘贴模拟依赖 Portal；Portal 不可用时降级为"仅写入剪贴板 + Toast '请手动 Ctrl+V 粘贴'"。
3. 若依赖 `ydotool`，安装包需要引导用户把当前用户加入 `input` 组。
4. 已知完全支持矩阵：X11 + i3/Openbox/Xfce4 → 完全可用；GNOME 45+ Wayland → 可用（Portal 齐备）；KDE Plasma 6 Wayland → 可用；其他组合 → 按启动检测结果展示能力。

## 权限申请的业务规则（跨平台通用）

1. **不静默失败**：任何系统权限缺失都必须有显式 UI 反馈。
2. **申请时机分两类**：
   - **核心权限（麦克风）**：首次启动向导 Step 2 **主动申请**——这是例外，因为不授权麦克风则整个应用无用武之地。详见 [onboarding.md](./onboarding.md)。
   - **其他权限（Accessibility / Input Monitoring / 通知）**：首次触发对应功能时申请；允许延后。
3. **运行时监测**：应用启动后启动一个权限监测 tick（每 10 秒查询一次已授权状态），捕获"运行中被撤销"的场景，立即更新 UI 状态和主界面横条。
4. **可重试**：设置 → 关于 → "检查系统权限" 按钮提供所有权限状态的实时面板，每项可重新申请。
5. **不绕过**：不尝试用任何技巧绕过系统权限限制（例如不使用 root、不使用 SUID）。
6. **降级路径明示**：所有"权限缺失时会降级"的功能，降级后必须通过 Toast / 横条让用户知道"当前是降级状态"，不沉默接受。
